# 模块化报告生成工作流（上下文隔离设计）

> 本文档定义 gold-ore skill 模式A/B/C 的模块化执行架构。
> **核心原则**：每个分析模块独立运行，只看自己领域的输入数据，防止跨模块上下文污染。

## 适用场景

当报告需要覆盖以下 3 个或以上独立模块时，启用本工作流：
- 地质分析（矿床类型、构造背景、成矿要素）
- 经济评价（NPV/IRR、品位-吨位、成本分析）
- 风险分析（政策、环保、社区、基建）

单模块任务（如仅做类型识别）不需要本工作流，直接执行对应 workflow。

---

## 架构设计

```
原始材料（用户提供）
       │
       ▼
┌─────────────────────────────────────────────────────┐
│  Step 0：材料分拣（主 Agent 执行，约 5 分钟）          │
│  - 扫描所有文件                                       │
│  - 按模块分类，生成 3 个独立的输入摘要文件             │
│  - 保存到 checkpoints/<session>/                     │
└─────────────────────────────────────────────────────┘
       │
       ├──────────────────────────────────────────────┐
       │                                              │
       ▼                                              ▼
┌──────────────────┐                    ┌─────────────────────┐
│  地质 Agent       │                    │  经济 Agent          │
│  只看：           │                    │  只看：              │
│  - 地质报告       │                    │  - 资源量数据        │
│  - 钻孔数据       │                    │  - 品位/吨位         │
│  - 地质图         │                    │  - 实时金属价格      │
│  - 岩相学描述     │                    │  - 可比项目成本      │
│  输出：geo.md     │                    │  输出：econ.md       │
└──────────────────┘                    └─────────────────────┘
       │                                              │
       └──────────────────┬───────────────────────────┘
                          │
                          ▼
              ┌─────────────────────┐
              │  风险 Agent          │
              │  只看：              │
              │  - 政策法规文件      │
              │  - 基建条件描述      │
              │  - 社区/环保信息     │
              │  - geo.md 摘要       │
              │  输出：risk.md       │
              └─────────────────────┘
                          │
                          ▼
              ┌─────────────────────┐
              │  整合 Agent          │
              │  只看：              │
              │  - geo.md            │
              │  - econ.md           │
              │  - risk.md           │
              │  （不看原始材料）    │
              │  输出：最终报告      │
              └─────────────────────┘
```

## Step 0：材料分拣（主 Agent 执行）

扫描用户工作目录，将文件分配到三个模块：

```python
# 分拣规则
GEO_KEYWORDS = ["地质", "钻孔", "岩心", "蚀变", "矿化", "构造", "地层", "岩相",
                 "geology", "drill", "core", "alteration", "mineralization"]
ECON_KEYWORDS = ["资源量", "储量", "品位", "吨位", "可研", "经济", "成本", "NPV",
                  "resource", "reserve", "grade", "tonnage", "feasibility", "cost"]
RISK_KEYWORDS = ["政策", "法规", "环保", "社区", "基建", "道路", "电力", "水源",
                  "policy", "regulation", "environment", "community", "infrastructure"]
```

分拣完成后，生成三个输入摘要文件：
- `checkpoints/<session>/geo_input.md`：地质模块的输入摘要（文件列表 + 关键数据点）
- `checkpoints/<session>/econ_input.md`：经济模块的输入摘要
- `checkpoints/<session>/risk_input.md`：风险模块的输入摘要

**重要**：摘要文件只包含该模块需要的信息，不包含其他模块的数据。

---

## Step 1：地质 Agent（独立上下文）

**输入**：仅 `geo_input.md` + `knowledge/deposit_types.md` + `knowledge/grade_tonnage.md`

**任务**：
1. 矿床类型识别（按 deposit_types.md 诊断特征矩阵）
2. 构造背景定位（按 global_belts.md）
3. 规模-品位评估（Tier 分级）
4. 成矿要素 10 项打分

**输出格式**（保存到 `checkpoints/<session>/geo.md`）：

```markdown
## 地质分析摘要

### 矿床类型
- 判定：[类型] (置信度: 高/中/低)
- 诊断证据：[3条独立证据]

### 构造背景
- 成矿带：[名称]
- 对比矿床：[3个同带矿床]

### 规模-品位
- Tier 分级：[Tier 1/2/3]
- 资源量估算：[X Mt @ Y g/t Au]
- 在累积频率曲线上的位置：[百分位]

### 成矿要素打分
| 要素 | 得分 | 依据 |
|------|------|------|
| ... | .../10 | ... |
**总分：XX/100**

### 地质结论
[2-3句话，不涉及经济和风险]
```

---

## Step 2：经济 Agent（独立上下文）

**输入**：仅 `econ_input.md` + `geo.md`（只读地质结论部分）+ 实时金属价格

**禁止读取**：原始地质报告、风险相关文件

**任务**：
1. 调用 mining-price-engine 获取实时金价
2. 基于 geo.md 的资源量数据执行 DCF 估值
3. 可比交易估值
4. 敏感性分析（金价±30%、OPEX±20%、CAPEX±20%）

**输出格式**（保存到 `checkpoints/<session>/econ.md`）：

```markdown
## 经济评价摘要

### 基础参数
- 金属价格：$X/oz（来源：yfinance，日期：YYYY-MM-DD）
- 资源量：[来自 geo.md]
- 采矿方式：[露天/地下]
- 年处理能力：X Mt/a

### DCF 估值
- NPV（8%折现率）：$X M
- IRR：X%
- 回收期：X 年

### 可比交易估值
- EV/Resource 倍数：$X/oz
- 估值区间：$X M - $Y M

### 加权估值
- DCF 60% + 可比交易 40% = $X M

### 敏感性分析
| 变量 | -30% | 基准 | +30% |
|------|------|------|------|
| 金价 | $X M | $Y M | $Z M |
| OPEX | ... | ... | ... |
| CAPEX | ... | ... | ... |

### 经济结论
[2-3句话，不涉及地质细节和风险]
```

---

## Step 3：风险 Agent（独立上下文）

**输入**：仅 `risk_input.md` + `geo.md`（只读矿床类型和地理位置）

**禁止读取**：经济数据、原始地质报告

**任务**：
1. 政策风险（矿权政策、外资限制、税收）
2. 环保合规风险（EIA 要求、尾矿处置）
3. 社区关系风险（原住民权益、搬迁）
4. 基础设施风险（道路、电力、水源）
5. 技术风险（选冶难度、难处理矿石）

**输出格式**（保存到 `checkpoints/<session>/risk.md`）：

```markdown
## 风险分析摘要

### 风险矩阵
| 风险类别 | 等级 | 主要因素 | 缓解措施 |
|----------|------|----------|----------|
| 政策风险 | 高/中/低 | ... | ... |
| 环保风险 | ... | ... | ... |
| 社区风险 | ... | ... | ... |
| 基建风险 | ... | ... | ... |
| 技术风险 | ... | ... | ... |

### 关键风险（Top 3）
1. [风险名称]：[具体描述] → [建议措施]
2. ...
3. ...

### 风险结论
[2-3句话，综合风险等级判断]
```

---

## Step 4：整合 Agent（只看三个摘要）

**输入**：仅 `geo.md` + `econ.md` + `risk.md`

**禁止读取**：任何原始材料、任何中间推理过程

**任务**：
1. 综合三个模块的结论
2. 检查内部一致性（地质结论与经济假设是否匹配）
3. 生成投资建议（推荐/谨慎/不推荐）
4. 按 `templates/project_assessment.md` 模板生成最终 DOCX

**一致性检查规则**：
- 地质 Tier 1 但经济 NPV 为负 → 标注"数据矛盾，需核查"
- 风险等级高但投资建议为推荐 → 必须在报告中明确说明风险溢价
- 品位数据在 geo.md 和 econ.md 中不一致 → 停止，告知用户

---

## 降级策略

如果并行 Agent 失败（API 错误、超时）：
1. 保存已完成模块的输出到 checkpoints
2. 降级为顺序执行：geo → econ → risk → 整合
3. 告知用户"已切换为顺序模式，已完成 X/4 个模块"

如果某个模块输出质量不达标（内容过短、缺少关键字段）：
1. 重新执行该模块（最多 1 次重试）
2. 重试仍失败则在最终报告中标注"[模块名]数据不完整，建议人工补充"
