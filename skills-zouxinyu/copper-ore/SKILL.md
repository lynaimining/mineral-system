# Copper Ore（铜矿）领域 skill

铜矿地质 / 找矿 / 成矿潜力分析的领域知识与工作流。**核心理念：类型识别 → 构造定位 → 规模—品位评估 → 成矿要素打分 → 预测靶区**。

## 3. 模式 A：铜矿项目分析（默认）

**目标**：基于用户提供的项目材料，给出"它是什么、有多大、好不好、还该做什么"的专业判断。

### 流程（模块化执行架构）

```
Step 0（前置检查）
    │
    ▼
Step 1（材料分拣）→ 生成 geo_input / econ_input / risk_input
    │
    ├────────────────────────────┐
    ▼                            ▼
Step 2-5（地质 Agent）      Step 6（经济 Agent）
  类型识别                    实时铜价获取
  构造定位                    DCF/NPV 估值
  规模-品位评估               可比交易估值
  成矿要素打分                敏感性分析
  输出: geo.md               输出: econ.md
    │                            │
    └────────────┬───────────────┘
                 ▼
         Step 7（风险 Agent）
           政策/环保/社区/基建/技术风险
           输出: risk.md
                 │
                 ▼
         Step 8（整合 + 质量审查）
           一致性检查
           双 Reviewer 审查
           输出: 最终报告 .docx
```

> **单模块任务**（如仅做类型识别）不启用模块化架构，直接执行对应步骤。

### Step 1：材料扫描与分拣

扫描用户工作目录中的：钻孔报告、地质图、化探数据、岩相学描述、年代学、同位素、PPT、可研。

**分拣规则**：
- 地质类：含"地质/钻孔/岩心/蚀变/矿化/构造/geology/drill/alteration"
- 经济类：含"资源量/储量/品位/可研/成本/NPV/resource/feasibility"
- 风险类：含"政策/法规/环保/社区/基建/policy/environment"

### Step 2：类型识别（决策树 — 决定后续所有分析）

```
容矿岩石是什么？
├─ 中酸性侵入体（花岗闪长岩/石英二长岩）
│   ├─ 有钾化-绢云母化-青磐岩化分带？
│   │   ├─ YES + Cu±Mo±Au → 🔴 Porphyry Cu
│   │   └─ NO → 检查矽卡岩接触带
│   │       ├─ 有石榴石-透辉石矽卡岩 → 🟡 Skarn Cu
│   │       └─ 无 → 需更多证据
│   └─ 共生 U+Au+REE + 铁氧化物蚀变？ → 🔴 IOCG
├─ 红色砂岩/碳酸盐岩
│   ├─ 有还原层（黑色页岩/蒸发岩）？
│   │   ├─ YES + Cu±Co±Ag → 🔴 SSC（沉积岩容矿）
│   │   └─ NO → 非 SSC
│   └─ 白云岩化 + 卤水通道？ → 🟡 可能 SSC
├─ 海相火山岩（双峰式）
│   ├─ 块状硫化物 + Cu-Zn-Pb？ → 🔴 VMS
│   └─ 非块状 → 检查其他类型
├─ 超镁铁-镁铁质侵入体
│   ├─ 硫化物矿物（Po-Pn-Ccp）+ Ni-Cu-PGE？ → 🔴 Magmatic Cu-Ni-PGE
│   └─ 无硫化物 → 非岩浆型
└─ 玄武岩（大陆溢流）
    └─ 自然铜 + 沸石化？ → 🟡 Native Cu
```

**诊断置信度判定**：
- 🔴 高置信度（≥3条独立诊断证据匹配）→ 直接进入后续分析
- 🟡 中置信度（2条证据匹配）→ 列出备选类型，继续但标注不确定性
- 🟢 低置信度（<2条证据）→ 告知用户信息不足，建议补充数据

### Step 3：构造定位
按 `knowledge/global_belts.md` 把项目放入全球成矿带（安第斯、特提斯、中非、SW Pacific、克拉通等），并对比同带头部矿床。

### Step 4：规模—品位评估
- 用 `knowledge/grade_tonnage.md` 的对应曲线，把项目的资源量/品位标在该类型的累积频率曲线上。
- 给出 Tier 1（>5 Mt Cu）/ Tier 2（1-5 Mt Cu）/ Tier 3（<1 Mt Cu）分级。
- 给出剥蚀深度判断（Sillitoe 模型：上覆—蚀变—矿化—下伏的顶底关系）。

### Step 5：成矿要素打分（10 项，每项 0-10 分）
按 `templates/project_assessment.md` 第 5 节的打分卡执行。10 项总分 ≥70 分为 Tier 1 候选，50-70 为 Tier 2，<50 为高风险。

### Step 6：经济性分析
- 获取实时铜价（LME/yfinance）
- DCF 估值（NPV@8%、IRR、回收期）
- 可比交易估值（EV/lb Cu）
- 敏感性分析（铜价±30%、OPEX±20%、CAPEX±20%、品位±15%）
- 盈亏平衡铜价计算

### Step 7：质量审查
按 `workflows/quality_review.md` 执行双 Reviewer 审查（详见第 9 节量化标准）。

### Step 8：输出
按 `templates/project_assessment.md` 模板输出 .docx。文件名 `Copper_Project_Assessment_<项目名>_<YYYYMMDD>.docx`。
输出位置按 CLAUDE.md 规定（矿业评估类→Desktop/02-矿业评估/）。

## 5. 模式 C：区域成矿潜力分析与预测（核心特色）

**触发**：用户说"成矿潜力"、"找矿预测"、"区划"、"靶区"、"远景"等。

### 流程（**矿床模型驱动 + 要素叠加**）
```
1. 区域类型选定（按用户提供的范围 + global_belts 匹配）
2. 选择控矿模型（按 deposit_types + tectonic_controls）
3. 列出关键控矿要素清单（exploration_criteria）
4. 各要素 GIS 化—权重—叠加（用户若有数据则量化；否则给出方法论）
5. 圈定 A/B/C 三级靶区
6. 给出钻探/化探/地球物理建议
7. 输出：报告 .docx + 打分表 .xlsx（如有数据）
```

### 找矿要素的标准化清单（每类型一套，详见 `knowledge/exploration_criteria.md`）
- **Porphyry**：弧岩浆带 / 中酸性侵入体 / 钾化—绢云母化蚀变 / Cu-Mo-Au 化探异常 / 磁高—电极化率高 / ZTEM 电阻率
- **SSC**：盆地裂谷期红层 / 还原层（黑色页岩、蒸发岩） / 卤水通道（断层）/ 白云岩化 / Cu-Co 异常
- **IOCG**：元古宙克拉通边缘 / 钠—钙—铁氧化物蚀变 / 大型磁异常 / 深部高重力 / U-Cu-Au-REE 化探
- **VMS**：海相双峰式火山岩 / 古热液喷口 / 半石化沉积盖层 / Cu-Zn-Pb 异常
- **Cu-Ni-PGE**：大火成岩省（LIP）/ 镁铁—超镁铁岩通道 / S 饱和证据 / 镍法精探

## 9. 量化审查标准（质量闭环核心）

### 9.1 双 Reviewer 审查矩阵

| 审查维度 | 🔴 不合格（必须修复） | 🟡 需改进（建议修复） | 🟢 合格 |
|----------|----------------------|----------------------|---------|
| 类型判定 | 类型判错或无证据支撑 | 证据不足（<3条） | ≥3条独立证据，排除法论证完整 |
| 构造定位 | 成矿带归属错误 | 对比矿床选择不当 | 成矿带正确，对比矿床≥3个 |
| 规模评估 | 品位/吨位数量级错误 | 未标注资源量类别 | Tier分级正确，数据来源清晰 |
| 经济分析 | NPV计算错误或无敏感性分析 | 参数假设未说明 | DCF+可比双法，敏感性完整 |
| 风险平衡 | 全面乐观无风险分析 | 风险列出但无缓解措施 | 风险+有利因素平衡，有缓解方案 |
| 数据来源 | 关键数据无来源标注 | 部分数据缺来源 | 所有数据标注来源和日期 |
| 图件质量 | 无图件或图件错误 | 图件<4张或标注不清 | ≥6张图件，标注完整 |
| 参考文献 | 引用虚假文献 | 格式不规范 | GB/T 7714格式，CrossRef验证 |

### 9.2 修复决策逻辑

```
统计 🔴 和 🟡 数量
    │
    ├─ 🔴 = 0 且 🟡 ≤ 2 → PASS，直接输出
    ├─ 🔴 = 0 且 🟡 > 2 → 修复 🟡 后输出（不需要重新审查）
    ├─ 🔴 ≥ 1 → 启动修复循环
    │   ├─ Round 1：修复所有 🔴 + 🟡
    │   ├─ 增量审查修复部分
    │   ├─ 仍有 🔴？
    │   │   ├─ YES → Round 2（最终轮）
    │   │   └─ NO → PASS
    │   └─ Round 2 后仍有 🔴？
    │       └─ 标记 NEEDS_HUMAN，输出报告 + 附录列出未解决问题
    └─ 特殊情况：Tier 1 项目（>5 Mt Cu）
        → 强制执行 2 轮审查，即使首轮全 🟢
```

## 10. 反模式清单（禁止事项）

### 10.1 地质分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| A1 | 仅凭单一特征判定矿床类型 | 多种类型可共享某些特征 | 至少3条独立诊断证据 + 排除法 |
| A2 | 把所有含铜矿化都归为斑岩型 | 斑岩型仅占铜矿的~60% | 严格按决策树逐步排除 |
| A3 | 忽略剥蚀深度对保存的影响 | 深剥蚀可能已移除矿体上部 | 必须评估 Sillitoe 剥蚀模型 |
| A4 | 用 Tier 1 矿床参数套用 Tier 3 项目 | 规模差异导致经济参数不可比 | 选择同 Tier 对比矿床 |
| A5 | 混用不同分类体系的资源量 | 331≠Measured，333≠Inferred | 明确标注分类体系，不混用 |

### 10.2 经济分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| B1 | 使用训练数据中的历史铜价 | 铜价波动大，历史价格不代表未来 | 必须获取实时 LME 铜价 |
| B2 | NPV 只用单一折现率 | 不同风险项目折现率不同 | 至少用 8%/10%/12% 三档 |
| B3 | 忽略选冶回收率对经济性的影响 | 回收率差10%可能决定项目生死 | 必须基于矿石类型估算回收率 |
| B4 | 不做敏感性分析 | 单一情景无法反映风险 | 铜价±30%、OPEX±20%、品位±15% |
| B5 | 单位换算错误（Mt↔kt↔t） | 数量级错误导致估值完全失真 | Python 计算器验证，双重检查 |

## 11. 置信度评分框架

每个关键结论必须附带置信度评分：

| 置信度 | 标记 | 条件 | 示例 |
|--------|------|------|------|
| **高** | ★★★ | ≥3条独立证据 + 与已知矿床高度一致 | "矿床类型为斑岩型（★★★）：钾化分带+Cu-Mo共生+花岗闪长岩容矿" |
| **中** | ★★☆ | 2条证据 或 部分数据缺失 | "规模可能达 Tier 2（★★☆）：品位数据充分但资源量仅为推断级" |
| **低** | ★☆☆ | <2条证据 或 关键数据缺失 | "经济可行性待验证（★☆☆）：缺少选冶试验数据" |
| **不确定** | ☆☆☆ | 无直接证据，纯推测 | "深部延伸潜力未知（☆☆☆）：无深部钻孔数据" |

**规则**：
- 每章结论必须标注置信度
- 总体投资建议的置信度 = 各章最低置信度
- ★☆☆ 或 ☆☆☆ 的结论必须在"信息缺口"章节列出获取方式

## 输出标准：Mineral System Assessment 统一模板

> **所有评估类输出（模式A/B/C）必须遵循统一模板**，详见：
> `~/.claude/skills/mineral-system-assessment-template.md`

模板包含9个模块（按需展开）：
1. 区域地质背景
2. 矿床类型判别（排除法推理+逐特征对比）
3. Mineral System七要素评估（逐项评分+证据+扣分项）
4. 全球类比矿床（相似点+差异点+对找矿的启示）
5. 成矿模型与找矿标志（垂向分带+走向分带+指示元素体系）
6. 靶区定位与优先级（三位一体+排序+逻辑说明）
7. 勘探方法论建议（有效手段+类比矿床经验+关键验证问题）
8. 三新提示（新矿种/新类型/新层位）
9. 多区域对比（批量评估模式时展开）

**质量红线**：
- 每个结论必须有证据支撑
- 类比必须具体（哪里像、哪里不像、意味着什么）
- 建议必须是"资深专家经验"，不是教科书常识
- 不给时间和预算估计
- 末尾必须有置信度声明
