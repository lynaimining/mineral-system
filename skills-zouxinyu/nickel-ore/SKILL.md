# Nickel Ore（镍矿）领域 skill

镍矿地质 / 找矿 / 成矿潜力分析的领域知识与工作流。**核心理念：类型识别（红土 vs 岩浆）→ 构造定位 → 规模—品位评估 → 冶炼工艺适配 → 成矿要素打分 → 预测靶区**。

### Step 1：材料扫描
扫描用户工作目录中的：钻孔报告、地质图、化探数据、岩相学描述、年代学、同位素、PPT、可研。

### Step 2：类型识别（决策树 — 决定后续所有分析）

按 `knowledge/deposit_types.md` 的"诊断特征矩阵"判定。

```
容矿岩石/环境是什么？
├─ 热带/亚热带风化壳（红色/褐色土壤覆盖）
│   ├─ 母岩为超镁铁岩（蛇纹岩/橄榄岩/辉石岩）？
│   │   ├─ YES → 检查垂向分带
│   │   │   ├─ 褐铁矿层（Fe>40%, Ni 0.8-1.5%）→ 🔴 Laterite-Limonite（适合 RKEF）
│   │   │   ├─ 腐泥土层（Mg<10%, Ni 1.2-2.5%）→ 🔴 Laterite-Saprolite（适合 HPAL）
│   │   │   └─ 混合型（无明显分带）→ 🟡 Laterite-Mixed（工艺需详细评估）
│   │   └─ NO → 非红土型镍矿
│   └─ 母岩为镁铁质岩？ → 🟡 可能为贫镍红土（品位通常不经济）
│
├─ 超镁铁-镁铁质侵入体（辉长岩/橄榄岩/辉石岩/苏长岩）
│   ├─ 含硫化物矿物（Po-Pn-Ccp）？
│   │   ├─ YES → 检查岩浆类型
│   │   │   ├─ 科马提岩（MgO>18%, 太古宙绿岩带）→ 🔴 Komatiite-hosted Ni
│   │   │   ├─ 大型层状侵入体 + Ni-Cu-PGE → 🔴 Magmatic Ni-Cu-PGE
│   │   │   ├─ 管道状/岩墙状侵入体 + 高 Ni/Cu 比 → 🔴 Conduit-type Ni-Cu
│   │   │   └─ 小型侵入体 + 低品位 → 🟡 需更多证据
│   │   └─ NO → 非岩浆硫化物型
│   └─ 有蛇纹石化但无硫化物？ → 检查是否为红土型母岩
│
├─ 黑色页岩/碳质沉积岩
│   ├─ Ni-Mo-V-U 富集？ → 🟡 Black Shale Ni（通常不经济，除非伴生 Mo/V）
│   └─ 无 → 非镍矿
│
└─ 热液蚀变带
    ├─ 与超镁铁岩相关的热液改造？ → 🟡 Hydrothermal Ni（少见，需验证）
    └─ 无超镁铁岩关联 → 非镍矿
```

**诊断置信度判定**：
- 🔴 高置信度（≥3条独立诊断证据匹配）→ 直接进入后续分析
- 🟡 中置信度（2条证据匹配）→ 列出备选类型，继续但标注不确定性
- 🟢 低置信度（<2条证据）→ 告知用户信息不足，建议补充数据

**关键诊断要素**：

**第一步：红土型 vs 岩浆型（最关键分叉）**
- **红土型（Laterite）**：热带风化壳，超镁铁岩（蛇纹岩/橄榄岩）风化产物，垂向分带（褐铁矿层/腐泥土层/蛇纹岩层）
- **岩浆型（Magmatic）**：岩浆硫化物，与超镁铁-镁铁质侵入体相关，硫化物矿物（镍黄铁矿/磁黄铁矿/黄铜矿）

**第二步：岩浆型细分**
- **Komatiite-hosted**：科马提岩（超镁铁质火山岩），太古宙绿岩带
- **Magmatic Ni-Cu-PGE**：镁铁-超镁铁质侵入体（辉长岩-橄榄岩），与 copper-ore 的 Cu-Ni-PGE 类型重叠

### Step 3：构造定位
按 `knowledge/global_belts.md` 把项目放入全球成矿带（环太平洋红土带、加拿大地盾、俄罗斯 Norilsk 等），并对比同带头部矿床。

### Step 4：规模—品位评估
- 用 `knowledge/grade_tonnage.md` 的对应曲线，把项目的资源量/品位标在该类型的累积频率曲线上。
- 给出 Tier 1（>1 Mt Ni）/ Tier 2（0.1-1 Mt Ni）/ Tier 3（<0.1 Mt Ni）分级。

### Step 5：冶炼工艺适配（镍矿特有）
按 `knowledge/processing_routes.md` 判定最适合的工艺路线：
- **HPAL**（高压酸浸）：适合高镁低铁红土矿（腐泥土层），产品为硫酸镍（电池级 Class 1）
- **RKEF**（回转窑-电炉）：适合低镁高铁红土矿（褐铁矿层），产品为镍铁（不锈钢级 Class 2）
- **Ferronickel**：传统火法冶炼，产品为镍铁
- **Flotation + Smelting**：岩浆硫化物型，浮选富集后冶炼，产品为电解镍（Class 1）

**关键判断**：
- 红土矿的 Ni/Fe 比、Mg/Fe 比决定工艺路线
- 电池级镍（Class 1）vs 不锈钢级镍（Class 2）的市场价差巨大

### Step 6：成矿要素打分（10 项，每项 0-10 分）
按 `templates/project_assessment.md` 第 5 节的打分卡执行。10 项总分 ≥70 分为 Tier 1 候选，50-70 为 Tier 2，<50 为高风险。

### Step 6b：经济性分析（镍矿特有）
- 获取实时镍价（LME NI=F / yfinance）
- **Class 1 vs Class 2 价差分析**：
  - Class 1（电池级，≥99.8% Ni）：电解镍、硫酸镍、MHP → 新能源需求驱动
  - Class 2（不锈钢级，NPI/FeNi）：镍铁、镍生铁 → 不锈钢需求驱动
  - 价差通常 $3000-8000/t，受电池需求波动影响
- DCF 估值（NPV@8%、IRR、回收期）
- 可比交易估值（EV/t Ni contained）
- 敏感性分析（镍价±30%、OPEX±20%、CAPEX±20%、品位±15%、回收率±10%）
- 盈亏平衡镍价计算
- **工艺成本对比**（关键）：
  - HPAL：CAPEX 高（$40,000-60,000/t Ni 年产能），OPEX 中等
  - RKEF：CAPEX 中等，OPEX 高（电力成本敏感）
  - 浮选+冶炼：CAPEX 低，但仅适用于岩浆硫化物型

### Step 7：质量审查
按 `workflows/quality_review.md` 执行双 Reviewer 审查（详见第 9 节量化标准）：
- Reviewer-1：地质技术准确性 + 类型↔工艺一致性
- Reviewer-2：逻辑与风险平衡 + 经济参数合理性
- 问题 severity 分级（Critical/High/Medium/Low）
- Tier 1 项目或用户要求高质量时，启动迭代修复循环（MAX_ROUNDS = 2）

### Step 8：输出
按 `templates/project_assessment.md` 模板输出 .docx。文件名 `Nickel_Project_Assessment_<项目名>_<YYYYMMDD>.docx`。
输出位置按 CLAUDE.md 规定（矿业评估类→Desktop/02-矿业评估/）。

## 5. 模式 C：区域成矿潜力分析与预测（核心特色）

**触发**：用户说"成矿潜力"、"找矿预测"、"区划"、"靶区"、"远景"等。

### 流程（**矿床模型驱动 + 要素叠加**）
```
1. 区域类型选定（红土带 vs 岩浆带）
2. 选择控矿模型（按 deposit_types + global_belts）
3. 列出关键控矿要素清单（exploration_criteria）
4. 各要素 GIS 化—权重—叠加（用户若有数据则量化；否则给出方法论）
5. 圈定 A/B/C 三级靶区
6. 给出钻探/化探/地球物理建议
7. 输出：报告 .docx + 打分表 .xlsx（如有数据）
```

### 关键产出（`templates/potential_prediction.md`）
- **潜力评分卡**：每 km² 网格 0-100 分
- **靶区清单**：A 级（>80 分，立即工程验证）、B 级（60-80，地表加密）、C 级（40-60，进一步勘查）
- **找矿模型简图**：含构造、岩性、蚀变、地化、地球物理标志
- **下一步工作建议**：钻孔/槽探/化探/地球物理优先级

- **图件**：成矿模式图、品位—吨位投影、地质平面、垂向分带剖面（红土型）、靶区图——优先用 SVG / PNG 嵌入 docx；不可幻觉数据。
- **数据可信度声明**：所有引用资源量、品位、坐标必须注明来源（公司年报、可研、文献）；模型参数注明来源文献。
- **不确定时**：宁可少说也不编造。AI 不能替代实测数据；建议用户用真实钻孔/化探/物探数据驱动。

## 9. 量化审查标准（质量闭环核心）

### 9.1 双 Reviewer 审查矩阵（8维）

| 审查维度 | 🔴 不合格（必须修复） | 🟡 需改进（建议修复） | 🟢 合格 |
|----------|----------------------|----------------------|---------|
| 类型判定 | 红土/岩浆类型判错或无证据支撑 | 证据不足（<3条）或未区分亚类 | ≥3条独立证据，红土亚类/岩浆亚类明确 |
| 构造定位 | 成矿带归属错误 | 对比矿床选择不当（规模/类型不匹配） | 成矿带正确，对比矿床≥3个且同类型 |
| 规模评估 | 品位/吨位数量级错误 | 未标注资源量类别（331/332/333 或 M/I/I） | Tier分级正确，数据来源清晰 |
| 工艺适配 | 工艺与矿石类型严重不匹配 | 未考虑 Ni/Fe 比或 Mg/Fe 比 | 工艺选择有化学数据支撑，产品等级明确 |
| 经济分析 | NPV计算错误或无敏感性分析 | 未区分 Class 1/Class 2 价差 | DCF+可比双法，Class 1/2 分别估值 |
| 风险平衡 | 全面乐观无风险分析 | 风险列出但无缓解措施 | 风险+有利因素平衡，有缓解方案 |
| 数据来源 | 关键数据无来源标注 | 部分数据缺来源或过时（>3年） | 所有数据标注来源和日期 |
| 图件质量 | 无图件或图件错误 | 图件<4张或缺少垂向分带剖面（红土型） | ≥6张图件，含分带剖面/成矿模式图 |

### 9.2 修复决策逻辑

```
统计 🔴 和 🟡 数量
    │
    ├─ 🔴 = 0 且 🟡 ≤ 2 → PASS，直接输出
    ├─ 🔴 = 0 且 🟡 > 2 → 修复 🟡 后输出（不需要重新审查）
    ├─ 🔴 ≥ 1 → 启动修复循环
    │   ├─ Round 1：修复所有 🔴 + 🟡
    │   ├─ 增量审查修复部分
    │   ├─ 仍有 🔴？
    │   │   ├─ YES → Round 2（最终轮）
    │   │   └─ NO → PASS
    │   └─ Round 2 后仍有 🔴？
    │       └─ 标记 NEEDS_HUMAN，输出报告 + 附录列出未解决问题
    └─ 特殊情况：Tier 1 项目（>1 Mt Ni）
        → 强制执行 2 轮审查，即使首轮全 🟢
```

## 10. 反模式清单（禁止事项）

### 10.1 地质分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| A1 | 不区分红土型和岩浆型直接分析 | 两者成因、经济参数、工艺路线完全不同 | Step 2 必须先完成类型识别决策树 |
| A2 | 红土矿不做垂向分带描述 | 褐铁矿层/腐泥土层/蛇纹岩层决定工艺选择 | 必须描述完整剖面分带及各层厚度/品位 |
| A3 | 把所有岩浆型都当 Norilsk 类比 | Norilsk 是极端特例（洪泛玄武岩+蒸发岩同化） | 按实际规模选择同 Tier 对比矿床 |
| A4 | 忽略蛇纹石化程度对红土矿品位的控制 | 蛇纹石化越完全，风化后 Ni 富集越好 | 必须评估母岩蛇纹石化程度 |
| A5 | 混用不同分类体系的资源量 | 331≠Measured，333≠Inferred | 明确标注分类体系，不混用 |

### 10.2 经济分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| B1 | 混淆红土型和岩浆型的经济参数 | OPEX差异巨大（HPAL $8000-12000/t vs 浮选 $4000-6000/t） | 必须按矿床类型选择对应的成本基准 |
| B2 | 忽略 Ni/Fe 比和 Mg/Fe 比对工艺选择的影响 | 这两个比值直接决定 HPAL vs RKEF 的适用性 | 必须计算并标注，作为工艺选择依据 |
| B3 | 把所有红土矿都当 HPAL 处理 | 高铁低镁矿只适合 RKEF，HPAL 酸耗过高不经济 | 按 Ni/Fe、Mg/Fe 比值判断适用工艺 |
| B4 | 不区分 Class 1 和 Class 2 产品定价 | 价差可达 $3000-8000/t，严重影响项目经济性 | 必须明确产品等级，分别定价 |
| B5 | 单位换算错误（Mt↔kt↔t，%↔ppm） | 数量级错误导致估值完全失真 | Python 计算器验证，双重检查 |

### 10.3 报告撰写反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| C1 | 全面乐观的"一边倒"结论 | 违反 CLAUDE.md 分析质量标准 | 风险+有利因素必须平衡 |
| C2 | 红土矿报告缺少垂向分带剖面图 | 分带是红土矿最核心的地质特征 | 必须包含完整的垂向分带剖面 |
| C3 | 工艺选择无化学数据支撑 | 工艺适配是镍矿评估的核心差异化步骤 | 必须基于 Ni/Fe/Mg/Co 数据选择工艺 |
| C4 | 编造不存在的参考文献 | 学术不端，严重损害可信度 | CrossRef 验证，无法确认则删除 |
| C5 | 忽略印尼/菲律宾出口禁令对项目的影响 | 政策风险是红土镍矿最大的非技术风险 | 必须评估所在国矿业政策和出口限制 |

## 11. 置信度评分框架

每个关键结论必须附带置信度评分：

| 置信度 | 标记 | 条件 | 示例 |
|--------|------|------|------|
| **高** | ★★★ | ≥3条独立证据 + 与已知矿床高度一致 | "矿床类型为红土型-腐泥土亚类（★★★）：垂向分带明确+Ni/Fe>0.1+母岩为蛇纹岩" |
| **中** | ★★☆ | 2条证据 或 部分数据缺失 | "适合 HPAL 工艺（★★☆）：Mg/Fe 比合适但缺少完整选冶试验" |
| **低** | ★☆☆ | <2条证据 或 关键数据缺失 | "经济可行性待验证（★☆☆）：缺少 CAPEX 估算和基建条件评估" |
| **不确定** | ☆☆☆ | 无直接证据，纯推测 | "深部延伸潜力未知（☆☆☆）：无深部钻孔数据" |

**规则**：
- 每章结论必须标注置信度
- 总体投资建议的置信度 = 各章最低置信度
- ★☆☆ 或 ☆☆☆ 的结论必须在"信息缺口"章节列出获取方式
- 工艺适配结论的置信度取决于化学分析数据的完整性：
  - 有完整 Ni/Fe/Mg/Co/SiO2 数据 → 可达 ★★★
  - 仅有 Ni 品位 → 最高 ★★☆
  - 无化学数据 → 最高 ★☆☆
