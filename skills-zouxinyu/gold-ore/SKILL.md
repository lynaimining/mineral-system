# Gold Ore（金矿）领域 skill

金矿地质 / 找矿 / 成矿潜力分析的领域知识与工作流。**核心理念：类型识别 → 构造定位 → 规模—品位评估 → 成矿要素打分 → 预测靶区**。

### 0.4 金矿类型识别决策树（Step 2 前置参考）

```
构造背景 + 容矿岩石 + 蚀变组合 → 类型判定
    │
    ├─ 造山带剪切带 + 绿片岩/角闪岩相变质岩
    │   ├─ 碳酸盐化-绢云母化-硅化 + Au-As-Sb-W？
    │   │   └─ YES → 🔴 Orogenic Gold（造山型）
    │   └─ 侵入体接触带 + Au-Bi-Te-W？
    │       └─ YES → 🟡 Intrusion-related Gold（侵入体相关型）
    │
    ├─ 碳酸盐岩台地/被动大陆边缘
    │   ├─ 脱碳酸盐化-硅化-黄铁矿化 + Au-As-Sb-Hg-Tl？
    │   │   └─ YES → 🔴 Carlin-type（卡林型）
    │   └─ 无典型 Carlin 蚀变？ → 检查其他类型
    │
    ├─ 火山弧/岛弧环境
    │   ├─ 明矾石-叶蜡石-地开石 + 高硫化？
    │   │   └─ YES → 🔴 Epithermal HS（高硫化浅成低温）
    │   ├─ 冰长石-绢云母 + 低硫化 + Au-Ag？
    │   │   └─ YES → 🔴 Epithermal LS（低硫化浅成低温）
    │   └─ 中酸性侵入体 + 钾化-绢云母化分带 + Au-Cu？
    │       └─ YES → 🔴 Porphyry Au-Cu（斑岩型）
    │
    ├─ 元古宙克拉通边缘
    │   ├─ 铁氧化物蚀变 + Cu-Au-U-REE？
    │   │   └─ YES → 🟡 IOCG-Au
    │   └─ 古砾岩层 + 碎屑黄铁矿 + 铀矿物？
    │       └─ YES → 🔴 Witwatersrand（古砾岩型）
    │
    └─ 海相火山岩序列
        └─ 块状硫化物 + Cu-Zn-Au？
            └─ YES → 🟡 VMS-Au
```

**诊断置信度判定**：
- 🔴 高置信度（≥3条独立诊断证据匹配）→ 直接进入后续分析
- 🟡 中置信度（2条证据匹配）→ 列出备选类型，继续但标注不确定性
- 🟢 低置信度（<2条证据）→ 告知用户信息不足，建议补充数据

## 3. 模式 A：金矿项目分析（默认）

**目标**：基于用户提供的项目材料（钻孔、化探、报告、PPT、影像），给出"它是什么、有多大、好不好、还该做什么"的专业判断。

### 流程
```
材料扫描 → 类型识别 → 构造定位 → 规模—品位评估 → 成矿要素打分（10 项） → 风险与建议 → 质量审查 → 输出
```

> **模块化执行（报告含 3+ 章节时启用）**：当报告需要覆盖地质分析、经济评价、风险分析三个模块时，
> 改用 `workflows/modular_report.md` 的上下文隔离架构：
> 材料分拣 → [地质 Agent | 经济 Agent] → 风险 Agent → 整合 Agent → 输出
> 每个 Agent 只看自己领域的输入，防止跨模块上下文污染。

### Step 1：材料扫描
扫描用户工作目录中的：钻孔报告、地质图、化探数据、岩相学描述、年代学、同位素、PPT、可研。

### Step 2：类型识别（决定后续所有分析）
按 `knowledge/deposit_types.md` 的"诊断特征矩阵"判定。**关键诊断要素**：
- **构造背景**：造山带剪切带→Orogenic；火山弧→Epithermal；碳酸盐岩台地→Carlin；克拉通盆地→Witwatersrand
- **蚀变组合**：碳酸盐化-绢云母化→Orogenic；明矾石-叶蜡石→HS Epithermal；成人石-绢云母→LS Epithermal；脱碳酸盐化-硅化→Carlin
- **共生元素**：Au-As-Sb-W→Orogenic；Au-As-Sb-Hg-Tl→Carlin；Au-Ag-As→Epithermal；Au-Cu→Porphyry/IOCG

### Step 3：构造定位
按 `knowledge/global_belts.md` 把项目放入全球成矿带（Yilgarn、Abitibi、Nevada、环太平洋等），并对比同带头部矿床。

### Step 4：规模—品位评估
- 用 `knowledge/grade_tonnage.md` 的对应曲线，把项目的资源量/品位标在该类型的累积频率曲线上。
- 给出 Tier 1（>100 t Au）/ Tier 2（10-100 t Au）/ Tier 3（<10 t Au）分级。

### Step 5：成矿要素打分（10 项，每项 0-10 分）
按 `templates/project_assessment.md` 第 5 节的打分卡执行。10 项总分 ≥70 分为 Tier 1 候选，50-70 为 Tier 2，<50 为高风险。

### Step 6：质量审查
按 `workflows/quality_review.md` 执行双 Reviewer 审查：
- Reviewer-1：地质技术准确性
- Reviewer-2：逻辑与风险平衡
- 问题 severity 分级（Critical/High/Medium/Low）
- Tier 1 项目或用户要求高质量时，启动迭代修复循环（MAX_ROUNDS = 2）

### Step 7：输出
按 `templates/project_assessment.md` 模板输出 .docx。文件名 `Gold_Project_Assessment_<项目名>_<YYYYMMDD>.docx`。
输出位置按 CLAUDE.md 规定（矿业评估类→Desktop/02-矿业评估/）。

## 5. 模式 C：区域成矿潜力分析与预测（核心特色）

**触发**：用户说"成矿潜力"、"找矿预测"、"区划"、"靶区"、"远景"等。

### 流程（**矿床模型驱动 + 要素叠加**）

## 9. 量化审查标准（质量闭环核心）

### 9.1 双 Reviewer 审查矩阵（8维）

| 审查维度 | 🔴 不合格（必须修复） | 🟡 需改进（建议修复） | 🟢 合格 |
|----------|----------------------|----------------------|---------|
| 类型判定 | 类型判错或无证据支撑 | 证据不足（<3条）或未排除备选类型 | ≥3条独立证据，排除法论证完整，置信度标注 |
| 构造定位 | 成矿带归属错误 | 对比矿床选择不当或数量不足 | 成矿带正确，对比矿床≥3个且同Tier |
| 规模评估 | 品位/吨位数量级错误或单位换算错误 | 未标注资源量类别（331/332/333 或 M/I/I） | Tier分级正确，数据来源清晰，单位明确 |
| 经济分析 | NPV计算错误/单位换算错误/无敏感性分析 | 参数假设未说明或金价非实时 | DCF+可比双法，敏感性完整，t↔oz换算已验证 |
| 风险平衡 | 全面乐观无风险分析（一边倒） | 风险列出但无缓解措施 | 风险+有利因素平衡，有缓解方案+概率评估 |
| 数据来源 | 关键数据无来源标注 | 部分数据缺来源或日期 | 所有数据标注来源、日期和资源量类别 |
| 图件质量 | 无图件或图件错误 | 图件<4张或标注不清 | ≥6张图件，标注完整，含成矿模式图 |
| 参考文献 | 引用虚假文献 | 格式不规范或缺少关键综述 | GB/T 7714格式，CrossRef验证，含类型权威综述 |

### 9.2 修复决策逻辑

```
统计 🔴 和 🟡 数量
    │
    ├─ 🔴 = 0 且 🟡 ≤ 2 → PASS，直接输出
    ├─ 🔴 = 0 且 🟡 > 2 → 修复 🟡 后输出（不需要重新审查）
    ├─ 🔴 ≥ 1 → 启动修复循环
    │   ├─ Round 1：修复所有 🔴 + 🟡
    │   ├─ 增量审查修复部分
    │   ├─ 仍有 🔴？
    │   │   ├─ YES → Round 2（最终轮）
    │   │   └─ NO → PASS
    │   └─ Round 2 后仍有 🔴？
    │       └─ 标记 NEEDS_HUMAN，输出报告 + 附录列出未解决问题
    └─ 特殊情况：Tier 1 项目（>100 t Au）
        → 强制执行 2 轮审查，即使首轮全 🟢
```

### 9.3 单位换算专项审查（金矿特有）

> **历史教训：2026-05-09 醴陵金矿估值因单位换算错误导致估值虚高6-8倍。此项为强制审查。**

| 检查项 | 验证方法 | 不通过则 |
|--------|----------|----------|
| t → oz 换算 | Python: `tonnes * 32150.7 = oz` | 🔴 必须修复 |
| oz → Moz 换算 | Python: `oz / 1_000_000 = Moz` | 🔴 必须修复 |
| EV/oz 计算 | 确认分子（USD M）分母（oz）单位一致 | 🔴 必须修复 |
| 货币换算 | USD M ↔ RMB 亿（M=10^6, 亿=10^8） | 🔴 必须修复 |
| 数量级 sanity check | 与同类型同Tier矿床对比是否合理 | 🟡 需说明 |

## 10. 反模式清单（禁止事项）

### 10.1 地质分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| A1 | 仅凭单一特征判定金矿类型 | 多种类型可共享某些特征（如Au-As组合见于Orogenic和Carlin） | 至少3条独立诊断证据 + 排除法 |
| A2 | 把所有剪切带金矿都归为造山型 | 侵入体相关型也产于剪切带，需区分Au-Bi-Te vs Au-As-Sb | 严格按决策树逐步排除，检查共生元素 |
| A3 | 忽略成矿时代与容矿岩石时代的关系 | 造山型金矿成矿晚于容矿岩石变质，Carlin型有特定时代窗口 | 必须评估成矿年代学证据 |
| A4 | 用 Tier 1 矿床参数套用 Tier 3 项目 | 规模差异导致经济参数不可比 | 选择同 Tier 对比矿床 |
| A5 | 混用不同分类体系的资源量 | 331≠Measured，333≠Inferred，换算需折减系数 | 明确标注分类体系，不混用 |

### 10.2 经济分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| B1 | 使用训练数据中的历史金价 | 金价波动大，历史价格不代表未来 | 必须获取实时 LBMA/COMEX 金价（$/oz） |
| B2 | NPV 只用单一折现率 | 不同风险项目折现率不同 | 至少用 5%/8%/10% 三档 |
| B3 | 忽略选冶回收率对经济性的影响 | 难选冶金矿（含砷/碳质）回收率可能仅60-70% | 必须基于矿石类型估算回收率 |
| B4 | 不做敏感性分析 | 单一情景无法反映风险 | 金价±20%、OPEX±20%、品位±15% |
| B5 | **单位换算错误（t↔oz↔Moz, USD M↔RMB亿）** | **数量级错误导致估值完全失真（历史教训：6-8倍偏差）** | **Python计算器强制验证，双重检查，逆向验算** |

| C1 | 全面乐观的"一边倒"结论 | 违反 CLAUDE.md 分析质量标准 | 风险+有利因素必须平衡 |

## 11. 置信度评分框架

每个关键结论必须附带置信度评分：

| 置信度 | 标记 | 条件 | 示例 |
|--------|------|------|------|
| **高** | ★★★ | ≥3条独立证据 + 与已知矿床高度一致 | "矿床类型为造山型（★★★）：剪切带控矿+碳酸盐化蚀变+Au-As-Sb-W组合" |
| **中** | ★★☆ | 2条证据 或 部分数据缺失 | "规模可能达 Tier 2（★★☆）：品位数据充分但资源量仅为333类" |
| **低** | ★☆☆ | <2条证据 或 关键数据缺失 | "经济可行性待验证（★☆☆）：缺少选冶试验数据，矿石含砷需确认" |
| **不确定** | ☆☆☆ | 无直接证据，纯推测 | "深部延伸潜力未知（☆☆☆）：无深部钻孔数据" |

**规则**：
- 每章结论必须标注置信度
- 总体投资建议的置信度 = 各章最低置信度
- ★☆☆ 或 ☆☆☆ 的结论必须在"信息缺口"章节列出获取方式
- 涉及单位换算的经济结论，必须标注"已Python验证"才可给 ★★★

## 输出标准：Mineral System Assessment 统一模板

> **所有评估类输出（模式A/B/C）必须遵循统一模板**，详见：
> `~/.claude/skills/mineral-system-assessment-template.md`

模板包含9个模块（按需展开）：
1. 区域地质背景
2. 矿床类型判别（排除法推理+逐特征对比）

- 不给时间和预算估计
- 末尾必须有置信度声明
