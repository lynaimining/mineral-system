# Lithium Ore（锂矿）领域 skill

锂矿地质 / 找矿 / 成矿潜力分析的领域知识与工作流。**核心理念：类型识别（硬岩 vs 盐湖）→ 构造定位 → 规模—品位评估 → 冶炼工艺适配 → 成矿要素打分 → 预测靶区**。

### Step 2：类型识别（决策树 — 决定后续所有分析）

```
容矿环境是什么？
├─ 伟晶岩/花岗岩体系
│   ├─ 含 Li-Cs-Ta 矿物组合？
│   │   ├─ YES → 检查锂矿物种类
│   │   │   ├─ 锂辉石（spodumene）为主 → 🔴 LCT Pegmatite（硬岩型-锂辉石）
│   │   │   ├─ 锂云母（lepidolite）为主 → 🟡 LCT Pegmatite（硬岩型-锂云母）
│   │   │   └─ 透锂长石（petalite）为主 → 🟡 LCT Pegmatite（硬岩型-透锂长石）
│   │   └─ NO → 检查 Nb-Y-F 组合
│   │       ├─ YES → ⚠️ NYF 型伟晶岩（不含经济锂矿化！）
│   │       └─ NO → 需更多证据
│   └─ 伟晶岩分带特征？
│       ├─ 完整分带（边缘→壁带→中间带→核部） → 高分异，利于锂富集
│       └─ 无分带/简单伟晶岩 → 低分异，锂品位可能不足
├─ 盐湖/干旱盆地环境
│   ├─ 有卤水层/蒸发岩序列？
│   │   ├─ YES → 检查 Li 浓度和 Mg/Li 比
│   │   │   ├─ Li > 200 mg/L 且 Mg/Li < 10 → 🔴 Brine（盐湖型-优质）
│   │   │   ├─ Li > 200 mg/L 且 Mg/Li 10-20 → 🟡 Brine（盐湖型-中等）
│   │   │   ├─ Li > 200 mg/L 且 Mg/Li > 20 → 🟡 Brine（盐湖型-高镁难选）
│   │   │   └─ Li < 200 mg/L → 🟢 次经济盐湖（需 DLE 技术突破）
│   │   └─ NO → 非盐湖型
│   └─ 地热卤水？
│       ├─ YES + Li > 100 mg/L → 🟡 Geothermal Brine（新兴类型）
│       └─ NO → 检查其他类型
├─ 粘土/沉积岩环境
│   ├─ 含锂粘土矿物（hectorite/jadarite/smectite）？
│   │   ├─ YES + Li > 1000 ppm → 🟡 Clay（粘土型）
│   │   └─ NO 或 Li < 1000 ppm → 非经济粘土型
│   └─ 火山凝灰岩风化壳？
│       ├─ YES → 🟡 Volcano-sedimentary Clay（火山沉积粘土型）
│       └─ NO → 非锂矿
└─ 其他
    └─ 无法归类 → 告知用户信息不足，建议补充数据
```

**诊断置信度判定**：
- 🔴 高置信度（≥3条独立诊断证据匹配）→ 直接进入后续分析
- 🟡 中置信度（2条证据匹配）→ 列出备选类型，继续但标注不确定性
- 🟢 低置信度（<2条证据）→ 告知用户信息不足，建议补充数据

### Step 3：构造定位
按 `knowledge/global_belts.md` 把项目放入全球成矿带（澳大利亚 Yilgarn、加拿大 Abitibi、南美锂三角等），并对比同带头部矿床。

### Step 4：规模—品位评估
- 用 `knowledge/grade_tonnage.md` 的对应曲线，把项目的资源量/品位标在该类型的累积频率曲线上。
- 给出 Tier 1（>1 Mt LCE）/ Tier 2（0.1-1 Mt LCE）/ Tier 3（<0.1 Mt LCE）分级。
- **LCE（Lithium Carbonate Equivalent）**：碳酸锂当量，统一单位

### Step 5：冶炼工艺适配（锂矿特有）
按 `knowledge/processing_routes.md` 判定最适合的工艺路线：

**硬岩型**：
- **锂辉石精矿路线**：浮选 → 焙烧 → 酸化 → 碳酸锂/氢氧化锂
- **锂云母路线**：硫酸法/石灰石焙烧法 → 碳酸锂
- **产品**：电池级碳酸锂（Li₂CO₃ ≥99.5%）、电池级氢氧化锂（LiOH·H₂O ≥56.5%）

**盐湖型**：
- **传统蒸发法**：蒸发池 → 碳酸锂（18-24 个月周期）
- **DLE（Direct Lithium Extraction）**：直接提锂（新技术，周期缩短至数周）
- **产品**：电池级碳酸锂

**关键判断**：
- 硬岩型品位要求：Li₂O > 1.0%（经济门槛）
- 盐湖型浓度要求：Li > 200 mg/L（经济门槛）
- Mg/Li 比（盐湖）：<10 为优质（Atacama），>20 为难选（青海）
- 电池级 vs 工业级价差：电池级价格 > 工业级 30-50%

### Step 6：成矿要素打分（10 项，每项 0-10 分）
按 `templates/project_assessment.md` 第 5 节的打分卡执行。10 项总分 ≥70 分为 Tier 1 候选，50-70 为 Tier 2，<50 为高风险。

### Step 7：质量审查
按 `workflows/quality_review.md` 执行双 Reviewer 审查：
- Reviewer-1：地质技术准确性
- Reviewer-2：逻辑与风险平衡
- 问题 severity 分级（Critical/High/Medium/Low）
- Tier 1 项目或用户要求高质量时，启动迭代修复循环（MAX_ROUNDS = 2）

### Step 8：输出
按 `templates/project_assessment.md` 模板输出 .docx。文件名 `Lithium_Project_Assessment_<项目名>_<YYYYMMDD>.docx`。
输出位置按 CLAUDE.md 规定（矿业评估类→Desktop/02-矿业评估/）。

## 5. 模式 C：区域成矿潜力分析与预测（核心特色）

**触发**：用户说"成矿潜力"、"找矿预测"、"区划"、"靶区"、"远景"等。

### 流程（**矿床模型驱动 + 要素叠加**）
```
1. 区域类型选定（硬岩带 vs 盐湖带）
2. 选择控矿模型（按 deposit_types + global_belts）
3. 列出关键控矿要素清单（exploration_criteria）
4. 各要素 GIS 化—权重—叠加（用户若有数据则量化；否则给出方法论）
5. 圈定 A/B/C 三级靶区
6. 给出钻探/化探/地球物理建议
7. 输出：报告 .docx + 打分表 .xlsx（如有数据）
```

### 关键产出（`templates/potential_prediction.md`）
- **潜力评分卡**：每 km² 网格 0-100 分
- **靶区清单**：A 级（>80 分，立即工程验证）、B 级（60-80，地表加密）、C 级（40-60，进一步勘查）
- **找矿模型简图**：含构造、岩性、蚀变、地化、地球物理标志
- **下一步工作建议**：钻孔/槽探/化探/地球物理优先级

## 9. 量化审查标准（质量闭环核心）

### 9.1 双 Reviewer 审查矩阵（8维）

| 审查维度 | 🔴 不合格（必须修复） | 🟡 需改进（建议修复） | 🟢 合格 |
|----------|----------------------|----------------------|---------|
| 类型判定 | 类型判错（如把NYF当LCT）或无证据支撑 | 证据不足（<3条）或未区分硬岩/盐湖子类型 | ≥3条独立证据，决策树路径完整 |
| 构造定位 | 成矿带归属错误 | 对比矿床选择不当（如用Atacama对比硬岩项目） | 成矿带正确，同类型对比矿床≥3个 |
| 规模评估 | LCE换算错误或品位/吨位数量级错误 | 未标注资源量类别（331/332/333或M/I/I） | Tier分级正确，LCE统一，数据来源清晰 |
| 工艺适配 | 工艺路线与矿石类型不匹配 | 未考虑Mg/Li比对盐湖工艺的影响 | 工艺路线合理，产品等级明确 |
| 经济分析 | NPV计算错误或使用2022年锂价高点 | 敏感性分析不含锂价±50%情景 | 实时锂价+多情景DCF+可比交易双法 |
| 风险平衡 | 全面乐观无风险分析 | 风险列出但无缓解措施或忽略DLE颠覆风险 | 风险+有利因素平衡，含技术颠覆评估 |
| 数据来源 | 关键数据无来源标注 | 部分数据缺来源或锂价未标注日期 | 所有数据标注来源和日期 |
| 图件质量 | 无图件或图件错误 | 图件<4张或缺少品位-吨位投影图 | ≥6张图件，含伟晶岩分带/盐湖剖面 |

### 9.2 修复决策逻辑

```
统计 🔴 和 🟡 数量
    │
    ├─ 🔴 = 0 且 🟡 ≤ 2 → PASS，直接输出
    ├─ 🔴 = 0 且 🟡 > 2 → 修复 🟡 后输出（不需要重新审查）
    ├─ 🔴 ≥ 1 → 启动修复循环
    │   ├─ Round 1：修复所有 🔴 + 🟡
    │   ├─ 增量审查修复部分
    │   ├─ 仍有 🔴？
    │   │   ├─ YES → Round 2（最终轮）
    │   │   └─ NO → PASS
    │   └─ Round 2 后仍有 🔴？
    │       └─ 标记 NEEDS_HUMAN，输出报告 + 附录列出未解决问题
    └─ 特殊情况：Tier 1 项目（>1 Mt LCE）
        → 强制执行 2 轮审查，即使首轮全 🟢
```

## 10. 反模式清单（禁止事项）

### 10.1 地质分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| A1 | 把所有伟晶岩都当锂矿 | NYF型伟晶岩不含经济锂矿化 | 必须先确认LCT vs NYF分类 |
| A2 | 混淆 Li₂O 品位和 LCE（碳酸锂当量） | 换算系数2.473，混用导致品位虚高/虚低 | 统一用LCE，明确标注换算：Li₂O × 2.473 = LCE |
| A3 | 忽略伟晶岩分带对品位分布的控制 | 锂矿化集中在中间带-核部，边缘带品位低 | 必须评估分带特征和矿化位置 |
| A4 | 用硬岩型参数评估盐湖型项目 | 两种类型的经济门槛、工艺、周期完全不同 | 严格按类型选择对比参数 |
| A5 | 忽略盐湖的水文地质条件 | 补给量决定可持续开采量，非仅看Li浓度 | 必须评估卤水补给-排泄平衡 |

### 10.2 经济分析反模式

| 编号 | 反模式 | 为什么错 | 正确做法 |
|------|--------|----------|----------|
| B1 | 用2022年锂价高点（50万+/吨）做NPV | 2024年锂价已跌至高点的1/5-1/4 | 必须用实时价格 + 敏感性分析（±50%） |
| B2 | 忽略 Mg/Li 比对盐湖经济性的决定性影响 | Mg/Li>20时除镁成本可能超过锂产品价值 | 盐湖项目必须评估Mg/Li比并量化除镁成本 |
| B3 | 不区分碳酸锂和氢氧化锂的价差 | 氢氧化锂溢价20-40%，影响项目收入测算 | 明确产品类型，分别定价 |
| B4 | 忽略DLE技术对盐湖项目估值的颠覆性影响 | DLE可能使高Mg/Li盐湖变为经济可采 | 必须评估DLE技术适用性和对项目估值的影响 |
| B5 | 单位换算错误（Li₂O↔LCE↔Li金属） | 数量级错误导致估值完全失真 | Python计算器验证：Li₂O×2.473=LCE，Li×5.323=LCE |

## 11. 置信度评分框架

每个关键结论必须附带置信度评分：

| 置信度 | 标记 | 条件 | 示例 |
|--------|------|------|------|
| **高** | ★★★ | ≥3条独立证据 + 与已知矿床高度一致 | "矿床类型为LCT伟晶岩（★★★）：锂辉石+Cs-Ta共生+完整分带+花岗岩母体" |
| **中** | ★★☆ | 2条证据 或 部分数据缺失 | "规模可能达 Tier 2（★★☆）：品位数据充分但资源量仅为推断级" |
| **低** | ★☆☆ | <2条证据 或 关键数据缺失 | "DLE工艺适用性待验证（★☆☆）：缺少中试数据" |
| **不确定** | ☆☆☆ | 无直接证据，纯推测 | "深部伟晶岩延伸潜力未知（☆☆☆）：无深部钻孔数据" |

**规则**：
- 每章结论必须标注置信度
- 总体投资建议的置信度 = 各章最低置信度
- ★☆☆ 或 ☆☆☆ 的结论必须在"信息缺口"章节列出获取方式
- 锂价相关结论最高为 ★★☆（因锂价波动性极大，无法高置信度预测）
