# Lithium-Ore Skill 质量审查流程

> 本文档定义 lithium-ore skill 的质量闭环机制，遵循 memory/skill-quality-loop.md 标准。

## 适用范围

- 模式 A（项目分析）：输出项目评价报告前必须执行
- 模式 B（报告撰写）：输出技术报告前必须执行
- 模式 C（潜力预测）：输出潜力评价报告前必须执行
- 模式 D（知识查询）：无需审查，直接回答

## 质量闭环配置

lithium-ore 采用 **A + F** 组合（最低配置）：
- **模式 A**：多视角并行审查（2 个 Reviewer）
- **模式 F**：问题 severity 分级

如果项目为 Tier 1 级别（>1 Mt LCE）或用户明确要求高质量，升级为 **A + C + F**：
- 增加 **模式 C**：迭代修复循环（MAX_ROUNDS = 2）

---

## 审查流程

### Step 1：双 Reviewer 并行审查

**Reviewer-1：地质技术准确性**
- 矿床类型判定是否正确（硬岩型 vs 盐湖��，对照 deposit_types.md 诊断特征）
- 冶炼工艺适配是否合理（锂辉石路线 vs 卤水路线，对照 processing_routes.md）
- 品位-吨位评估是否符合该类型曲线（对照 grade_tonnage.md）
- 成矿要素打分是否有数据支撑
- 参考文献引用是否准确

**Reviewer-2：逻辑与风险平衡**
- 结论是否存在"一边倒"乐观倾向（CLAUDE.md 防一边倒规则）
- 风险分析是否充分（必须包含主要风险和不确定因素）
- 冶炼工艺选择的经济性分析是否合理（电池级 vs 工业级产品价差）
- 数据来源是否标注清晰
- 假设条件是否明确
- 建议是否可操作

### Step 2：问题 Severity 分级（模式 F）

| Severity | 定义 | 示例 | 处理 |
|----------|------|------|------|
| Critical | 致命错误，会导致误判 | 矿床类型判错（硬岩误判为盐湖）、工艺路线完全不适配、品位数量级错误 | 必须修复 |
| High | 重大问题，影响可信度 | 缺少风险分析、数据无来源、关键假设未说明、工艺经济性分析缺失 | 必须修复 |
| Medium | 中等问题，影响专业性 | 术语不准确、参考文献格式错误、图表标注不清 | 建议修复 |
| Low | 轻微问题，不影响结论 | 措辞优化、排版细节 | 可忽略 |

### Step 3：修复决策

```
IF (Critical + High) == 0:
  → PASS，输出最终报告
ELSE IF 项目为 Tier 1 或用户要求高质量:
  → 启动迭代修复循环（模式 C，MAX_ROUNDS = 2）
  → 修复 Critical + High 问题
  → 增量审查修复部分
  → 若仍有 Critical/High 且超过 2 轮，标记 NEEDS_HUMAN
ELSE:
  → 修复 Critical 问题（High 问题列入报告附录"待完善事项"）
  → 输出报告
```

## 锂矿特有的审查要点

### 硬岩型锂矿（LCT Pegmatite）
- **伟晶岩分带识别**：边缘带 → 壁带 → 中间带 → 核心带是否正确判定
- **锂矿物识别**：锂辉石 vs 锂云母 vs 透锂长石
- **共生元素**：Ta-Nb-Cs-Be 是否评估副产价值
- **品位要求**：Li₂O > 1.0%（经济门槛）是否满足

### 盐湖型锂矿（Brine）
- **Mg/Li 比判定**：<10 为优质、10-20 为中等、>20 为难选
- **气候条件**：是否位于干旱气候带（年降雨量 <200 mm）
- **卤水深度**：是否满足经济开采要求（10-200 m）
- **DLE 技术适用性**：是否评估直接提锂技术可行性

### 工艺经济性
- **产品等级**：电池级（Li₂CO₃ ≥99.5%）vs 工业级价差是否考虑
- **工艺成本**：硬岩（CAPEX 中等、OPEX 中等）vs 盐湖（CAPEX 高、OPEX 低）差异是否分析
- **建设周期**：硬岩（2-3 年）vs 盐湖传统蒸发（3-5 年）vs DLE（1-2 年）是否考虑
- **环境风险**：硬岩尾矿处理、盐湖水资源消耗是否提及

## 与 critical-metals / mineral-system-eval 的协作

当用户任务同时涉及 lithium-ore 和其他 skill 时：
1. **关键金属框架**：lithium-ore 可以被 critical-metals（规划中）调用
2. **通用框架**：mineral-system-eval 负责五要素框架评分
3. **质量审查**：lithium-ore 的 Reviewer-1 + mineral-system-eval 的 Reviewer 共同完成
