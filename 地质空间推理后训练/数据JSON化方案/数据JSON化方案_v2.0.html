1项目定位

定位声明：本方案是 MSM（Mineral System Model）基座大模型的数据准备阶段，为后训练提供高质量、结构化的地质领域训练数据。

最终产品形态

给定目标区域 → 输出地质背景分析 + 矿种预测 + Mineral System Model 完整报告

区域地质输入 → MSM 基座模型推理 → 结构化报告输出
邹心宇职责：Phase A 数据准备全流程，交付结构化训练数据给后训练工程师。包括：数据采集、版面解析、知识抽取、样本合成、Spatial Embedding 构建。

3数据处理管线 M1–M6

M1

版面解析

DocLayout-YOLO 检测页面元素 + GOT-OCR 2.0 端到端识别。输出：段落/表格/图片/公式的 bbox 坐标与类型标签。

DoD: layout F1 ≥ 0.92
M2

文本OCR + 公式识别

多语言OCR引擎处理正文文本，LaTeX公式识别引擎处理数学表达式。输出：结构化文本流 + 公式LaTeX。

DoD: CER ≤ 1.5%
M3

表格抽取 + 单位归一

表格结构识别 → 单元格内容提取 → 单位标准化（g/t, %, ppm统一映射）。输出：结构化表格JSON + 归一化数值。

DoD: cell-F1 ≥ 0.88
M4

图件分类 + Caption解析

地质图件分类（地质图/剖面图/地球化学图/显微照片等）→ 图注提取与结构化。输出：图件类型标签 + 结构化caption。

DoD: macro-F1 ≥ 0.80
M5

知识图谱：实体 + 关系抽取

地质实体识别（矿床/岩体/构造/矿物/地层）→ 关系抽取（控矿/赋存/切割/侵入）→ 知识图谱构建。

DoD: entity-F1 ≥ 0.78
M6

训练样本合成（S1–S4）

基于M1-M5产出，合成4类训练样本：长文本预训练(S1)、多模态VQA(S2)、空间推理trace(S3)、RL偏好对(S4)。

DoD: 见Section 7产出规格

4统一数据契约 MSM-DocSchema v1

所有处理模块的输出统一遵循此Schema，确保数据互操作性和下游训练管线的无缝对接。

5博士论文核心创新

关键洞察：博士论文 ≠ 更长的期刊论文。博士论文是"完整的推理教程"——包含从问题提出、多尺度观察、证据链构建到结论推导的全过程，是训练模型空间推理能力的最佳素材。

数据类型说明训练用途预期产出

knowledge_structure 系统性知识框架：章节逻辑、论证结构 长文本组织能力 ~500套/500篇
contrastive_cases 对比分析：成矿vs非成矿、不同矿床类型对比 S4 RL偏好对构建 ~3,000对/500篇

6Spatial Embedding 层（核心创新）

核心理念：地质推理的本质是空间推理。Spatial Embedding层将地质概念、空间关系、尺度信息编码为可计算的向量表示，使模型具备"空间感知"能力。
◆

概念空间向量库

512维语义向量 + 128维空间属性向量。覆盖矿床类型、岩石组合、构造样式、蚀变类型等核心概念。

512+128维
★

空间关系图谱

分层架构：区域尺度(>100km) → 矿区尺度(1-10km) → 矿体尺度(<1km) → 矿物尺度(um级)。编码控矿要素的空间拓扑关系。

多尺度空间索引

基于H3六边形网格的多分辨率索引。支持从全球尺度到矿体尺度的快速空间查询与关联检索。

基于FAISS的高效向量检索。支持"找到与目标区域地质特征最相似的已知矿床"类查询。
