博士论文 — 核心创新

💡 关键洞察：博士论文 ≠ 更长的期刊论文

博士论文的独特价值：完整的推理链条（"观察 → 背景整合 → 假设检验 → 推断 → 验证"），多尺度因果关联，推理过程透明。

期刊论文的局限：篇幅限制，只展示最终结论和关键证据，推理过程被压缩，很多细节被删掉。

对训练的意义：期刊论文教模型"快速判断"，博士论文教模型"推理链条"。

推理案例提取
提取跨章节的完整推理案例，包含观察、背景整合、假设检验、推断、验证五个步骤，直接生成CoT格式QA对。
多尺度关联
提取"同一现象在不同尺度的表现"，训练模型的尺度转换能力（显微→露头→矿床→区域）。
知识图谱骨架
从文献综述章节构建"矿床类型→典型矿床→关键特征"的知识图谱框架。
对比辨析案例
从讨论章节提取"为什么是A而不是B"的辨析内容，生成评估基准题。
博士论文的多种用法
用法数据来源训练阶段价值
推理链条训练 reasoning_cases SFT 教模型完整推理过程（CoT格式）
多尺度关联 scale_linking_cases SFT 训练尺度转换能力
知识图谱骨架 knowledge_structure GraphRAG 构建矿床类型知识框架
DPO偏好对齐 reasoning_cases DPO 作为chosen样本（详细推理）
评估基准 contrastive_cases Evaluation 生成对比推理题
系统性知识 chapters CPT 全文喂入建立知识底座
关键决策记录
决策点选择理由
存储粒度 每文档每层一个JSON 并行友好，单篇失败不影响全局
矿床类型标注 Layer 1就标注 方便按类型筛选，MVP阶段关键
引用关系 简单存字符串数组 重点是论文内部空间关系
细粒度标注 Layer 2做 Layer 1保持Marker原始输出
钻孔数据 CSV/Parquet单独存 JSON会臃肿，表格数据用表格格式
资源量分类 保留原文 不同标准不完全等价，强行映射丢失信息
博士论文处理 独立Schema，提取推理案例 核心价值是推理链条，不是信息量
分阶段使用策略
训练阶段博士论文用法期刊论文用法
CPT 全文喂入（包括文献综述），建立系统性知识框架 全文喂入，覆盖广度

数据配比建议
CPT阶段 (1-2B tokens)
• 期刊论文: 60%
• JORC/NI 43-101: 30%
• 书籍: 10%
• 博士论文: 全文喂入（不单独计入配比）
SFT阶段 (8-15K QA对)
• 期刊论文: 50%（快速判断类）
• 博士论文: 30%（推理链条类）
• JORC: 15%（定量预测类）
• 书籍: 5%（教学式QA）
DPO阶段 (5-10K偏好对)
• 博士论文推理段落 vs 期刊论文简化版本: 70%
• Opus vs Sonnet生成对: 30%

实施路线图

Phase 1: MVP验证（Week 1-2）

1. 精选50篇论文（斑岩Cu 25篇 + 造山Au 25篇）
2. 提取为Layer 1 JSON
3. 生成200条QA对
4. LoRA微调验证效果
5. Go/No-Go决策

Phase 2: 全量处理（Week 3-7）

CPT (1-2B tokens) → LoRA SFT (8-15K QA对) → DPO对齐 (5-10K偏好对) → LazyGraphRAG搭建 → MSM-Bench评估
