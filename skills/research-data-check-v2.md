# 研报数据核查 Skill（v2.0 — 交叉验证版）

这个 Skill 专门用于核查研究报告（特别是金融、大宗商品市场研报）中的数据时效性和准确性。v2.0 新增**多源交叉验证**能力：对每个数据点搜索 ≥3 个独立来源进行比对，自动检测数据矛盾，并标注四级置信度（高/中/低/矛盾）。

## 核心功能

1. **自动识别数据源**: 提取文档中的所有数据源引用
2. **时效性检查**: 识别过时数据（如2024年），搜索2025年最新版本
3. **白名单验证**: 只接受头部机构（IEA、BloombergNEF、高盛等）的数据
4. **🆕 多源交叉验证**: 每个数据点搜索 ≥3 个独立来源，比对一致性，标注置信度
5. **🆕 矛盾检测与分析**: 源间数据冲突时自动分析可能原因并记录各方数值
6. **直接修改文档**: 在原Word文档中更新数据并标注来源+置信度
7. **标注待核查**: 找不到数据的标注"待人工核查"

## 工作流程

### 1. 获取文档路径

- 首先询问用户 Word 文档的完整路径
- 确认文档是 .docx 格式
- 如果用户已提供路径，直接使用

### 2. 文档解析与分析

使用 Read 工具读取文档后，进行以下分析：

#### 数据源识别
自动识别以下模式：
- `数据源: IEA Global EV Outlook 2024`
- `(来源: BloombergNEF, 2024)`
- `根据World Gold Council数据...`
- 年份标记：2024、2023等
- 核查标记：\"核查\"、\"待验证\"、\"需更新\"

#### 可信数据源白名单

只接受以下机构的数据：

**能源与气候**
- IEA (国际能源署)
- IRENA (国际可再生能源机构)
- BloombergNEF (彭博新能源财经)

**金属与矿业**
- CRU Group
- S&P Global Market Intelligence
- Wood Mackenzie
- Johnson Matthey (铂族金属)
- USGS (美国地质调查局)

**贵金属**
- World Gold Council (世界黄金协会)
- LBMA (伦敦金银市场协会)
- CPM Group, Metals Focus

**金融投资**
- Goldman Sachs (高盛)
- Morgan Stanley (摩根士丹利)
- J.P. Morgan, McKinsey, BCG, Bain

**国际组织**
- IMF, World Bank, OECD
- 国家统计局、美联储

**交易所**
- LME, SHFE, COMEX, ICE

**其他来源**: 自动标注"⚠ 待人工核查 [原因: 数据源不在白名单]"

### 4. 网络搜索验证（多源交叉验证）

对每个数据点使用 WebSearch 工具，**必须搜索 ≥3 个独立来源**：

#### 搜索策略（四轮递进）
```
第1轮: "[原始机构名] [报告名] 2025"          ← 原始来源最新版
第2轮: "[同领域第二机构] [数据主题] 2025"     ← 独立交叉源
第3轮: "[同领域第三机构] [数据主题] 2025"     ← 第三独立源
第4轮（矛盾时）: 权威仲裁源（如 USGS/IMF/World Bank）
```

#### 搜索示例
```
原数据: "2024年全球铜产量2100万吨 (来源: USGS)"

第1轮: "USGS Mineral Commodity Summaries 2025 copper production"
第2轮: "ICSG copper production 2025 world"
第3轮: "CRU Group global copper output 2025"
第4轮（如矛盾）: "World Bank commodity markets copper production latest"
```

#### 交叉验证来源矩阵

| 数据领域 | 首选来源 | 交叉源1 | 交叉源2 | 仲裁源 |
|----------|----------|---------|---------|--------|
| 铜/锌/铅产量 | USGS | ICSG | CRU Group | World Bank |
| 黄金市场 | World Gold Council | LBMA | Metals Focus | IMF |
| 锂/钴/镍 | USGS | S&P Global | Benchmark Mineral | IEA |
| 能源/电动车 | IEA | BloombergNEF | IRENA | World Bank |
| 宏观经济 | IMF | World Bank | OECD | 国家统计局 |
| 价格数据 | LME/COMEX | S&P Global | CRU | Bloomberg |

#### 验证逻辑
```
找到2025年数据 → ✓ 更新数据
找到2024年Q4数据 → ✓ 部分更新
只找到2024年数据 → 保持原数据，标注"已验证"
未找到数据 → ⚠ 标注"待人工核查"
数据源不在白名单 → ⚠ 标注"数据源需验证"
```

### 4.5 交叉验证与置信度评定（🆕 v2.0）

在完成多源搜索后，对每个数据点执行以下比对流程：

#### 置信度判定规则

```
决策树:
├─ ≥3 个独立来源数值一致（偏差 < 一致阈值）→ 置信度: 高 🟢
├─ 2 个独立来源数值一致 → 置信度: 中 🟡
├─ 仅 1 个来源可获取 → 置信度: 低 🟠
└─ 来源间数值矛盾（偏差 > 矛盾阈值）→ 置信度: 矛盾 🔴
    └─ 触发第4轮搜索（仲裁源）
    └─ 记录各来源数值 + 分析矛盾原因
```

#### 偏差阈值标准

| 数据类型 | 一致阈值（视为相同） | 矛盾阈值（标红） |
|----------|---------------------|------------------|
| 产量/储量（历史数据） | <5% | >10% |
| 价格（现货/历史） | <3% | >8% |
| 预测数据（未来值） | <15% | >25% |
| 增长率/占比 | <3个百分点 | >5个百分点 |

#### 矛盾分析模板

当置信度为 🔴 矛盾时，必须分析原因：
```
矛盾数据分析:
- 数据点: [具体数据]
- 来源A: [机构] = [数值] (发布日期: YYYY-MM)
- 来源B: [机构] = [数值] (发布日期: YYYY-MM)
- 偏差: [X%]
- 可能原因:
  □ 统计口径不同（如含/不含再生铜）
  □ 时间窗口不同（日历年 vs 财年）
  □ 地理范围不同（全球 vs 不含中国）
  □ 预测假设不同（供给侧 vs 需求侧模型）
  □ 数据更新时滞（一方尚未纳入最新修正）
- 建议采用: [推荐来源] (理由: [更权威/更新/口径更匹配])
```

#### 置信度汇总统计

核查完成后输出置信度分布：
```
📊 交叉验证统计:
━━━━━━━━━━━━━━━━━━
🟢 高置信度 (≥3源一致): X 个 (XX%)
🟡 中置信度 (2源一致):  X 个 (XX%)
🟠 低置信度 (仅1源):   X 个 (XX%)
🔴 矛盾 (源间冲突):    X 个 (XX%)
━━━━━━━━━━━━━━━━━━
整体数据可靠性: [高/中/低]
```

## 特殊情况处理

### 付费数据库
- BloombergNEF、Wood Mackenzie等需付费
- 策略：搜索公开摘要、新闻稿、第三方引用
- 无法获取时标注需订阅

### 矛盾数据
- 多个来源数据不一致时（v2.0 自动检测）
- 按偏差阈值判定：<一致阈值视为相同，>矛盾阈值标红
- 触发第4轮仲裁源搜索
- 分析矛盾原因（统计口径/时间窗口/地理范围/预测假设/更新时滞）
- 标注所有来源和差异百分比
- 给出建议采用来源及理由

### 预测数据
- 明确标注为"预测值"
- 注明预测的基准时间和假设条件
- 对比不同机构的预测差异

### 中文vs英文来源
- 优先使用原始语言版本
- 注意翻译可能的偏差
- 国内数据优先使用中文官方来源
